# Implementation Plan: cdd-storage

This microservice persists the artifacts generated by `cdd-engine` and serves them to `cdd-docs-ui` and `cdd-publisher`.

- [x] **1. Scaffolding**
  - [x] Initialize standard Rust binary project.
  - [x] Setup `actix-web` for fast artifact ingress and egress.
- [x] **2. Storage Backend Interface**
  - [x] Define an `ArtifactStore` trait with methods: `put`, `get`, `delete`.
  - [x] Implement the `LocalDiskStore` backend for immediate development ease.
  - [ ] (Optional) Implement the `S3Store` using the `aws-sdk-s3` crate.
- [x] **3. Ingestion API**
  - [x] Implement a `POST /upload/{org_id}/{repo_id}/{version}` endpoint.
  - [x] Secure this endpoint so that only internal services (like `cdd-engine`) can write to it.
- [x] **4. Retrieval API (Phase 2)**
  - [x] Implement a `GET /artifact/{org_id}/{repo_id}/schema.json` endpoint.
  - [x] Ensure this endpoint is blazingly fast and supports HTTP caching headers (ETag, Last-Modified) so `cdd-docs-ui` loads instantly.
  - [x] Implement endpoints to download the full zipped SDK source code.

## Quality Standards
- [x] No use of `unwrap`
- [x] No use of `anyhow` or similar approaches, instead have one big error enum (with `derive_more`)
- [x] Maintain 100% test coverage
- [x] Maintain 100% doc coverage
